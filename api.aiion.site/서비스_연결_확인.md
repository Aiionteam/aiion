## 게이트웨이 설정 확인

### 라우팅 규칙
모든 서비스는 Spring Cloud Gateway(포트 8080)를 통해 라우팅됩니다.

**예시:**
- `http://localhost:8080/user/**` → `user-service:8082`
- `http://localhost:8080/diary/**` → `diary-service:8083`
- `http://localhost:8080/chatbot/**` → `aihoyun-chatbot-service:9001`

### CORS 설정
게이트웨이에서 CORS를 통합 관리:
- 허용된 Origin: `http://localhost:3000`, `http://frontend:3000`, `http://127.0.0.1:3000`
- 허용된 메서드: GET, POST, PUT, DELETE, OPTIONS, PATCH
- 허용된 헤더: 모든 헤더 (`*`)

## Docker Compose 구조

### 서비스 빌드 경로
모든 마이크로서비스는 `../service.aiion.site`에서 빌드됩니다:
```yaml
build:
  context: ../service.aiion.site
  dockerfile: ./{service-name}/Dockerfile
```

### 네트워크
모든 서비스는 `spring-network` 브리지 네트워크에 연결되어 서로 통신할 수 있습니다.

## 서비스 실행 확인 방법

### 1. Docker Compose로 전체 서비스 실행
```bash
cd api.aiion.site
docker-compose up -d
```

### 2. 개별 서비스 상태 확인
```bash
docker-compose ps
```

### 3. 게이트웨이 헬스 체크
```bash
curl http://localhost:8080/actuator/health
```

### 4. 게이트웨이 라우팅 확인
```bash
curl http://localhost:8080/actuator/gateway/routes
```

### 5. 서비스별 헬스 체크
```bash
# User Service
curl http://localhost:8082/actuator/health

# Diary Service
curl http://localhost:8083/actuator/health

# Auth Service
curl http://localhost:8087/actuator/health
```

## 문제 해결

### 서비스가 시작되지 않는 경우
1. Docker 로그 확인:
   ```bash
   docker-compose logs {service-name}
   ```

2. 데이터베이스 연결 확인:
   - PostgreSQL이 실행 중인지 확인
   - Redis가 실행 중인지 확인

3. 포트 충돌 확인:
   ```bash
   netstat -ano | findstr :8080
   ```

### 게이트웨이에서 서비스에 접근하지 못하는 경우
1. 서비스가 같은 네트워크에 있는지 확인:
   ```bash
   docker network inspect api-aiion-site_spring-network
   ```

2. 서비스 이름 확인:
   - Docker Compose에서 서비스 이름과 컨테이너 이름이 일치하는지 확인

3. 게이트웨이 로그 확인:
   ```bash
   docker-compose logs gateway
   ```
