## 전체 개요

본 시스템은 **마이크로서비스 아키텍처**를 기반으로 하며, **Spring Cloud Gateway**를 통한 통합 게이트웨이 패턴을 사용합니다.

### 핵심 원칙
- ✅ **단일 진입점**: 모든 요청은 API Gateway(8080)를 통해 라우팅
- ✅ **서비스 분리**: 도메인별 독립적인 마이크로서비스
- ✅ **언어 다양성**: Spring Boot (Java) + FastAPI (Python)
- ✅ **통합 관리**: 하나의 게이트웨이에서 모든 라우팅 관리

## 레이어 구조

### 1. 프론트엔드 레이어 (www.aiion.site)
- **기술**: Next.js 14+ (React)
- **포트**: 3000
- **역할**: 
  - 사용자 인터페이스 제공
  - API Gateway(8080)로 모든 요청 전송
  - JWT 토큰 관리 및 전달

### 2. Edge Layer (게이트웨이)
- **기술**: Spring Cloud Gateway (Spring Boot 3.x)
- **포트**: 8080
- **역할**:
  - 모든 요청의 단일 진입점
  - 라우팅 및 로드 밸런싱
  - CORS 처리
  - 요청/응답 변환

### 3. AI 서비스 레이어 (ai.aiion.site)
- **기술**: Python FastAPI
- **서비스**:
  - `chatbot-service` (9001): AI 챗봇, 텍스트 분류, 일기 검색
  - `weather-service` (9004): 기상청 API 연동
  - `crawler-service` (9003): 웹 크롤링 및 데이터 수집

### 4. 마이크로서비스 레이어 (api.aiion.site)
- **기술**: Spring Boot 3.x (Java 21)
- **서비스**:
  - `diary-service` (8083): 일기 관리
  - `calendar-service` (8084): 캘린더 및 이벤트 관리
  - `user-service` (8082): 사용자 관리
  - `auth-service` (8087): 인증 및 권한 관리
  - `common-service` (8081): 공통 기능
  - `soccer-service` (8085): 축구 정보
  - `culture-service` (8086): 문화 정보
  - `healthcare-service` (8088): 건강 정보
  - `account-service` (8089): 계정 관리

### 5. 데이터 레이어
- **PostgreSQL** (5432): 관계형 데이터베이스
- **Redis** (6379): 인메모리 데이터베이스 (캐싱, 세션)

## API 엔드포인트

### AI 서비스 엔드포인트

#### 챗봇 서비스 (`/chatbot/**`)
- `POST /chatbot/chat`: 챗봇 대화
- `POST /chatbot/classify`: 텍스트 분류

#### 날씨 서비스 (`/weather/**`)
- `GET /weather/mid-forecast`: 중기예보 조회
- `GET /weather/short-forecast`: 단기예보 조회
- `GET /weather/regions`: 지원 지역 목록

#### 크롤러 서비스 (`/crawler/**`)
- `GET /crawler/bugsmusic`: 벅스 실시간 차트
- `GET /crawler/danawa_tv`: 다나와 TV 상품 목록
- `GET /crawler/netflix`: Netflix 영화 목록
- `GET /crawler/movie`: KMDB 영화 정보

### 마이크로서비스 엔드포인트

#### 일기 서비스 (`/diary/**`)
- `GET /diary/diaries/user`: JWT 기반 일기 조회
- `POST /diary/diaries`: 일기 생성
- `PUT /diary/diaries`: 일기 수정
- `DELETE /diary/diaries`: 일기 삭제

#### 캘린더 서비스 (`/calendar/**`)
- `GET /calendar/events/user`: 이벤트 조회
- `POST /calendar/events`: 이벤트 생성
- `GET /calendar/tasks/user`: 작업 조회
- `POST /calendar/tasks`: 작업 생성

#### 인증 서비스 (`/auth/**`, `/oauth2/**`)
- `POST /auth/login`: 로그인
- `POST /auth/register`: 회원가입
- `POST /auth/refresh`: 토큰 갱신
- `GET /oauth2/authorization/{provider}`: OAuth2 인증

#### 사용자 서비스 (`/user/**`)
- `GET /user/{id}`: 사용자 정보 조회
- `PUT /user/{id}`: 사용자 정보 수정

## 보안

### 인증/인가
- **JWT (JSON Web Token)**: 모든 인증된 요청에 사용
- **OAuth2**: 소셜 로그인 지원
- **토큰 검증**: 각 서비스에서 JWT 검증 수행

### 데이터 보안
- **사용자별 데이터 격리**: JWT에서 userId 추출하여 데이터 필터링
- **CORS**: Spring Cloud Gateway에서 통합 관리
- **HTTPS**: 프로덕션 환경에서 적용 (현재 개발 환경은 HTTP)

## 주요 특징

### ✅ 통합 게이트웨이
- 모든 요청이 단일 진입점(8080)을 통해 라우팅
- CORS, 인증, 로깅 등 공통 기능 중앙 관리

### ✅ 서비스 간 통신
- AI 서비스 → 마이크로서비스: API Gateway를 통한 HTTP 통신
- 마이크로서비스 → 데이터베이스: 직접 연결

### ✅ 확장성
- 각 서비스 독립적으로 스케일링 가능
- 새로운 서비스 추가 시 Gateway 라우팅만 추가

### ✅ 유지보수성
- 서비스별 독립적인 배포 및 업데이트
- 명확한 책임 분리
