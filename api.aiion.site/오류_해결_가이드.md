# api.aiion.site 오류 해결 가이드

## 발견된 오류

### Linter 오류
```
api.aiion.site/service/culture-service:
  - Project 'culture-service' is missing required source folder: 'src/test/java'
  - Project 'culture-service' is missing required source folder: 'src/main/java'
  - The project cannot be built until build path errors are resolved
```

## 원인 분석

이 오류는 **IDE 캐시 문제**입니다. 

1. `service` 디렉토리는 이미 `service.aiion.site`로 이동되었습니다.
2. `api.aiion.site/settings.gradle`에는 service 참조가 없습니다 (정상).
3. 하지만 IDE가 이전 경로(`api.aiion.site/service`)를 캐시하고 있습니다.

## 해결 방법

### 방법 1: IDE 캐시 정리 (권장)

#### VS Code / Cursor
1. **Command Palette** 열기 (`Ctrl+Shift+P`)
2. `Java: Clean Java Language Server Workspace` 실행
3. IDE 재시작

#### IntelliJ IDEA / Eclipse
1. **Invalidate Caches / Restart** 실행
2. IDE 재시작

### 방법 2: Gradle 캐시 정리

```bash
cd api.aiion.site

# Gradle 빌드 캐시 정리
./gradlew clean

# Gradle 데몬 중지
./gradlew --stop

# .gradle 디렉토리 삭제 (선택사항)
Remove-Item -Recurse -Force .gradle
```

### 방법 3: 프로젝트 재로드

1. IDE에서 프로젝트 닫기
2. `api.aiion.site` 디렉토리 다시 열기
3. Gradle 프로젝트로 인식되도록 대기

## 확인 사항

### ✅ 정상 상태 확인

1. **service 디렉토리 없음 확인**
   ```bash
   # service 디렉토리가 없어야 정상
   Test-Path api.aiion.site/service  # False
   ```

2. **settings.gradle 확인**
   ```gradle
   // service 참조가 없어야 정상
   rootProject.name = 'spring-server'
   include 'server'
   include 'server:gateway'
   ```

3. **Gradle 빌드 확인**
   ```bash
   ./gradlew tasks
   # 오류 없이 실행되어야 함
   ```

## 현재 상태

- ✅ `api.aiion.site`에는 `server` 모듈만 포함
- ✅ `service.aiion.site`에 모든 마이크로서비스 포함
- ✅ `docker-compose.yaml`은 올바른 경로(`../service.aiion.site`) 참조
- ⚠️ IDE 캐시에 이전 경로가 남아있음 (무해하지만 오류 표시)

## 결론

이 오류는 **실제 빌드나 실행에는 영향을 주지 않습니다**. 
- Gradle 빌드는 정상 작동합니다.
- Docker Compose는 올바른 경로를 참조합니다.
- 단지 IDE가 이전 경로를 기억하고 있을 뿐입니다.

**해결**: IDE 캐시를 정리하거나 무시해도 됩니다.
