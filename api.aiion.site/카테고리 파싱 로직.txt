1. 카테고리 정의
- 최상위 카테고리: 일기, 건강, 가계, 문화, 공부
- 일기 안에서도 세부 카테고리로 교차 태깅 가능 (예: "오늘 스트레칭 했다" → 일기 + 건강)

2. 데이터 준비
- 사용자 입력 텍스트(일기, 메모 등)를 수집.
- 각 문장/단락을 카테고리별로 라벨링 → 멀티라벨 데이터셋 구축.
- 예: "오늘 스트레칭 했다" → ["일기", "건강"]
- "결제 API 만졌다" → ["일기", "가계"]

3. 전처리
- 문장 단위 분리.
- 형태소 분석/토큰화 (예: Mecab, KoNLPy).
- 불용어 제거, 키워드 추출.

4. 모델 선택
- 멀티라벨 분류 모델 사용:
- 간단히 시작: Logistic Regression, Naive Bayes, SVM (scikit-learn의 OneVsRestClassifier).
- 고도화: KoBERT, KLUE 같은 한국어 Transformer 모델 → 각 카테고리 확률 출력.

5. 파싱 로직
- 입력 텍스트 → 문장 단위로 모델에 넣음.
- 각 카테고리별 확률 계산.
- Threshold(예: 0.3 이상) 넘으면 해당 카테고리로 태깅.
- 항상 일기는 기본 포함.

6. 출력 구조 예시
{
  "일기": true,
  "건강": ["스트레칭", "11시간 수면"],
  "가계": ["결제 API"],
  "문화": [],
  "공부": []
}



7. 후처리 & UX
- 결과를 태그/뱃지로 표시 → 사용자가 직관적으로 확인.
- 문장 하이라이트: "이 문장이 건강 관련으로 분류됨".
- 수정 기능: 사용자가 직접 카테고리 교정 → 재학습 데이터로 활용.
